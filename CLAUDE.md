# Code Style Guidelines

## Variable Naming Rules
- Use `camelCase` for all local variables and function parameters.
- Keep names descriptive (e.g., use `userId`, not just `u` or `id`).
- Avoid abbreviations unless they are standard acronyms (e.g., use `httpResponse`, not `httpResp`).

## Function Naming Rules
- Use `camelCase` for all private functions and internal helper methods.
- Use `PascalCase` only for exported/public functions.
- Always start function names with a clear action verb (e.g., `getUserData`, `calculateTotal`, `validateEmail`).
